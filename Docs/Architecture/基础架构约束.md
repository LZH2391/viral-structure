# 通用架构约束

## 定位

本文档是爆款结构迁移引擎的早期通用架构约束。它不规定固定目录、固定模块或固定领域对象，只提供后续开发必须持续遵守的底线原则。

`PlanMD` 只用于阶段计划和讨论结果；长期工程约束以 `Docs/Architecture` 下的文档为准。

## 基本原则

### 1. 先保持可理解，再追求完整

早期功能可以快速试错，但每个核心能力都要能说清楚：

- 它解决什么问题。
- 它接收什么输入。
- 它产出什么结果。
- 它依赖哪些上游结果。
- 它会影响哪些下游能力。

如果一个功能说不清边界，先补说明，再写实现。

### 2. 数据流要能追踪

核心结果不能只是一段临时文本或一个散落的变量。至少要能追踪：

- 结果来自哪个用户输入或样例。
- 经过了哪些关键步骤。
- 使用了哪些重要参数、策略或模型版本。
- 当前结果和旧结果是什么关系。

不要求一开始建立完整 artifact 系统，但实现时必须给未来追踪、返工和对比留出位置。

### 3. 返工不要破坏历史

用户不满意并重新生成时，默认不应静默覆盖旧结果。优先保留旧结果，并记录新结果基于什么产生。

早期可以用轻量版本字段、父级引用或历史记录实现；后续再根据实际复杂度演进为正式版本、分支或 artifact 体系。

### 4. 变化要有明确位置

高频变化不要散落在各处条件判断里。遇到可能频繁调整的内容，优先考虑是否应该放在：

- 配置中。
- 策略中。
- prompt/template 版本中。
- schemaVersion 或 payload 扩展中。
- 独立模块或服务边界中。

不要求一开始抽象完整，但要避免让一个变化点牵连无关代码。

### 5. 边界要服务功能，不提前画死

模块、目录、领域对象都可以随着功能演进调整。当前不强制固定分层，但要避免明显混乱：

- 展示逻辑、模型调用、业务推理不要无边界地混在一起。
- 下游逻辑不要随意修改上游结果。
- 临时实现可以存在，但要能被识别和替换。
- 新增核心能力时，应说明它和现有能力的关系。

### 6. 调试能力是基础能力

核心流程必须能定位问题。至少要能回答：

- 哪一次操作出问题。
- 哪一步出问题。
- 输入和输出摘要是什么。
- 是否调用了模型。
- 失败是否可重试。

具体日志、trace、debug snapshot 规则以 `AGENTS.MD` 和 `viral-structure-observability-reviewer` 为准。

## 当前建议概念

以下概念是当前阶段的建议，不是强制源码结构：

- `Project`：一次创作工作区。
- `Stage`：一个关键处理步骤。
- `Result` 或 `Artifact`：某一步产生的结果。
- `Version`：同类结果的不同版本。
- `Branch`：从某个结果继续尝试的新方向。
- `Trace`：一次运行或调试链路。

实现时可以使用更轻的名字或结构，但不要丢失这些概念背后的责任：工作区、步骤、结果、历史、分支、追踪。

### 代码文件粒度约束

- 单个业务文件原则上不超过 200 行。
- 超过 200 行时，必须说明为什么暂不拆分。
- 超过 300 行默认视为需要拆分。
- 一个文件只回答一个问题。
- UI 文件只负责布局和交互，不承载业务流程、模型调用、媒体处理或存储逻辑。
- 核心流程文件只负责编排，不直接实现具体媒体处理、模型调用、存储细节。
- 同一文件内如果同时出现 UI、业务判断、外部调用、状态持久化中的三类，必须拆分。


## 早期审查问题

新增或修改核心功能时，优先回答：

- 这个功能的输入和输出是否清楚？
- 它是否让结果更难追踪？
- 它是否会覆盖用户可能想找回的历史结果？
- 它是否把模型调用、展示、业务推理混在了一起？
- 它最可能变化的部分在哪里，后续是否容易调整？
- 出问题时，日志和调试信息能否定位到具体步骤？

这些问题用于发现风险，不用于阻止合理的早期探索。

## 明确禁止

- 禁止把所有核心逻辑写成不可追踪的一次性脚本。
- 禁止静默覆盖用户可能需要对比或恢复的核心结果。
- 禁止让模型调用散落到无法统一记录、替换和调试的位置。
- 禁止普通日志裸写敏感内容、完整 prompt 或完整素材内容。
- 禁止长期保留无人能解释的临时结构。

## 审查分工

- 架构演进风险由 `viral-structure-architecture-reviewer` 检查。
- 日志、调试、trace、debug snapshot 和模型调用留痕由 `viral-structure-observability-reviewer` 检查。
- 早期审查以发现风险、提出问题和输出任务卡为主，不以固定架构形状卡死实现。
