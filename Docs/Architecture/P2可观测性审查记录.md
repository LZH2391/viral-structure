## P2 Findings

1. 音频抽取失败虽然不会阻断样例处理，但降级原因没有进入 `sample.audio.extracted` 的 stage 输出，也没有受控 DebugSnapshot。
2. 顶层 API 异常在未进入业务 stage 时只返回安全 JSON，缺少 request 级 trace，无法在 `/debug` 中检索。
3. DebugSnapshot 默认使用 `snapshot_${stageId}.json`，同一 stage 多次写入会覆盖旧快照。

## 验收标准

- `sample.audio.extracted` 的 `stage.end` 输出 `available`、`degraded`、`reason`、`debugSnapshotUri`。
- 音频抽取降级 snapshot 只包含命令摘要、stderr 摘要、exitCode、mediaOperation、retryable。
- 顶层 API catch 生成 `api.request.handle` 的 `stage.start` 与 `stage.fail`，并写 request 级 DebugSnapshot。
- 400 类请求错误也写入可检索 fail，响应仍保持安全 JSON。
- `writeDebugSnapshot` 默认生成 `snapshot_${stageId}_${shortUniqueId}.json`，同一 `stageId` 连续写入不会覆盖。
