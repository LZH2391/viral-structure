# 可观测性审查基线

## Baseline

- baseline commit: `835dfaa`
- 记录时间：2026-05-21
- 本轮范围：只修复前端本地 stage 持久化、音频波形解码失败可追踪、Worker 失败语义明确化这 3 个 P1。

## P1 Findings

1. 前端 UI stage 只写入 React state，刷新后不可审计，且没有进入统一 `/debug` 页面。
2. 音频波形解码失败以空 peaks 降级为占位图，缺少 `stage.fail` 与 DebugSnapshot。
3. `audioPeaks.worker.ts` 使用 `peaks: []` 同时表达无音频、环境不可用和解码失败，失败语义不可追踪。

## 验收标准

- 前端 UI trace 使用 `uiTraceId`，不得伪造后端 `traceId`。
- `POST /api/debug/ui-events` 将 `stage.start / stage.end / stage.fail` 写入 `Runtime/DebugSnapshots/{uiTraceId}.log.jsonl`。
- `/api/debug/traces` 与 `/api/debug/traces/:traceId` 同时支持 `trace_*` 和 `uiTrace_*`。
- 音频波形最终没有生成 peaks 时，必须先写 `stage.fail` 和 DebugSnapshot，再展示 placeholder。
- Worker 响应必须区分 `{ ok: true, peaks }` 与 `{ ok: false, peaks: [], error: { code, message, retryable } }`。
