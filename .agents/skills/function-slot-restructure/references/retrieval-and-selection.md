# 检索与选择

当需要从多视频库中选择槽位和原子时，使用本参考。

## 选择顺序

按以下顺序选择：

1. **canonical slot chain / 标准槽位链**：需要哪些观众状态跃迁
2. **slot variants / 槽位变体**：哪些源样例最匹配每个所需槽位
3. **script atoms / 脚本原子**：哪种主张实现适合目标
4. **rhythm atoms / 节奏原子**：哪种注意力模式适合主张和时长
5. **packaging atoms / 包装原子**：哪种证明/视觉实现适合可用素材
6. **bindings and rules / 绑定与规则**：什么必须同步、承接或避免

不要先检索包装样式，再强行围绕它拼槽位。

## 候选评分

可用定性判断或脚本评分。遵循以下优先级：

### 1. 功能匹配

候选槽位是否产生目标观众状态跃迁？

高匹配通常满足：

- `viewerStateBefore` 和 `viewerStateAfter` 匹配链路中的目标位置
- `persuasionTask` 匹配目标任务
- 槽位所需同步点可实现

### 2. 主张与证明匹配

候选的证明需求是否能被目标素材满足？

示例：

- 机制主张需要机制解释或视觉证明
- 操作主张需要步骤提示和完成动作
- 结果主张需要与前置关切绑定的结果证据
- 长期信任主张需要时间证据、使用痕迹、评价、日志或重复反馈

### 3. 节奏匹配

节奏是否支撑信息量？

示例：

- 快速连击适合痛点激活，不适合复杂机制
- 稳定高密度适合解释
- 停顿后动作适合步骤到结果的转场
- 慢速证言适合信任收束

### 4. 包装匹配

包装功能是否匹配主张和目标生产资源？

先选证明功能，再选视觉样式。

示例：

- 问题定位 -> 近景、高亮、光标圈选、裁切、对比框
- 机制 -> 图解、覆盖层、屏幕标注、演示剖面
- 步骤 -> 图标、倒计时、清单、手势、界面指针
- 结果 -> 前后对比、输出屏、近景、数字变化
- 信任 -> 记录、重复证明、使用痕迹、证言、评价、收据、使用日志

### 5. 可靠性

优先选择：

- 更高 `confidence`
- 无 `needReview`
- 在多个样例中有重复支持的 slot type
- 被多个样例重复支持的规则
- 清楚的 source refs

### 6. 多样性

除非用户要求忠实变体，否则避免完全从一个源样例构建新视频。

可优先混合：

- 来自高支持模板的槽位链
- 来自相近品类的 script atom
- 来自相似时长/风格的 rhythm atom
- 来自拥有正确证明资产样例的 packaging atom

## 检索模式

### 精确槽位检索

当用户要求某个具体 `slotType` 时使用。

输出：

- top candidate slot variants
- 来源样例
- 脚本/节奏/包装选项
- 证明要求
- 风险

### 链路检索

当用户要求制作一个新视频时使用。

输出：

- 推荐槽位链
- 为什么适合目标
- 从库中借用了哪些槽位
- 因库中缺口而生成或插入了哪些槽位

### 缺口感知检索

当 corpus 缺少样例时使用。

输出：

- 可用的库候选
- 缺失的槽位或原子类型
- 生成式 fallback 实现
- 置信度降级说明

## 槽位混合规则

可以混合不同视频的 script、rhythm、packaging atoms，前提是：

- 它们共享相同或兼容的 slot type
- script 的 proof need 能被 packaging atom 满足
- rhythm 未在 `avoidFor` 中排斥该 script claim type
- 必要同步点可以对齐
- 跨槽位承接保持完整

不要仅因为标签听起来相似就混合原子。

## 选择解释示例

```text
选择 sample_014 的 problem_activation，因为它的观众状态跃迁匹配目标开场，并且对象-动作同步很强。节奏改用 sample_006，因为目标是 12 秒视频，需要更快进入。包装替换为 sample_021 的屏幕录制高亮，因为目标是 SaaS，不是护肤。绑定校验通过：问题对象、光标高亮和点击动作可以落在同一拍点。
```
