# 输出格式

使用以下格式保持响应一致。

## A. 语料库审计输出

```markdown
# 语料库审计

## 范围
- 检测到的样例数：
- slot variants：
- canonical slot types：
- atom variants：
- templates：
- rules/bindings：

## 强覆盖区域
...

## 稀疏区域
...

## 规则强度
| 规则/功能 | 支持度 | 置信度 | 备注 |
|---|---:|---:|---|

## 建议的库改进
...
```

## B. 槽位检索输出

```markdown
# 槽位检索

## 目标理解
...

## 候选槽位
| 所需槽位 | 选中候选 | 来源 | 选择理由 | 风险 |
|---|---|---|---|---|

## 替代候选
...

## 缺失或弱覆盖
...
```

## C. 重组输出

```markdown
# 重组方案

## 1. 目标理解
[品类、受众、目标、假设]

## 2. 选中槽位链
| 顺序 | slot type | 操作 | 来源/支持 | 角色 |
|---:|---|---|---|---|

## 3. 逐槽位方案
### 槽位 1：[slot type]
- 脚本角色：
- 节奏角色：
- 包装/证明角色：
- 同步点：
- 选中来源 variants：
- 可替换变量：

## 4. 脚本草案 / 节拍表
[可用的分段脚本]

## 5. 节奏曲线
[快/稳/停顿/峰值/收束曲线]

## 6. 包装方案
[视觉证明和覆盖层说明]

## 7. 绑定审计
| binding | 状态 | 备注 | 必要修复 |
|---|---|---|---|

## 8. 变体
- A：
- B：
- C：
```

## D. 校验与修复输出

```markdown
# 校验与修复

## 映射
| 脚本/分镜部分 | 映射槽位 | 置信度 |
|---|---|---:|

## 问题
| 严重度 | 问题 | 破坏的规则 | 修复 |
|---|---|---|---|

## 修复后链路
...

## 修复后脚本/方案
...
```
