# 类剪映结构迁移工作台总计划

## 1. 总体定位

第一版采用桌面工作台形态，参考剪映/CapCut 的视频编辑器心智：素材区、预览区、属性区、时间线、顶部工具栏。

但本项目不是普通剪辑软件，而是结构迁移工作台：

- 左侧不是单纯素材库，而是素材、样例结构、结构库、版本历史。
- 中间不是单纯播放器，而是样例预览、结构标注、新方案预览和对比预览。
- 底部时间线不只是剪辑时间线，而是视频时间线、结构段落线和迁移映射线。
- 右侧不只是参数面板，而是当前片段理解、结构解释、迁移建议和编辑控制。

项目核心仍然是：从优质样例视频中拆解可迁移的创作结构，再迁移到用户的新主题、商品或素材中。

## 2. 主界面布局

采用五区布局：

```text
顶部工具栏
├─ 项目名 / 保存状态 / 运行状态 / 导出入口 / 设置

左侧资源区      中央预览区              右侧属性区
├─ 素材          ├─ 视频预览              ├─ 当前片段信息
├─ 样例结构      ├─ 结构标注浮层          ├─ AI 理解结果
├─ 结构库        ├─ 样例/新方案切换        ├─ 迁移解释
└─ 版本历史      └─ 对比预览              └─ 参数与重跑

底部时间线
├─ 视频帧轨
├─ 字幕/语音轨
├─ 结构段落轨
└─ 迁移映射轨
```

默认第一屏状态：

- 左侧打开“素材”。
- 中间显示上传区或样例视频预览。
- 右侧显示处理状态和下一步提示。
- 底部时间线为空，上传处理完成后展示抽帧和时间戳。

## 3. 阶段流程

### 3.1 样例视频输入与处理

用户上传或选择一条样例视频。

系统需要生成：

- 原始视频引用。
- 标准化视频引用。
- 封面帧。
- 抽帧结果。
- 音频轨。
- 视频基础元信息。
- 处理状态。
- 后续视频理解入口。

本阶段只做输入和基础处理，不做爆款结构判断。

### 3.2 视频理解与结构拆解

系统基于抽帧、时间戳、音频、字幕等信息理解样例视频。

第一版理解方式：

```text
帧级理解 -> 片段级理解 -> 全局时序理解 -> 结构卡
```

重点拆解：

- 开头 hook。
- 内容段落。
- 镜头节奏。
- 字幕布局。
- 画面包装。
- 卖点推进。
- 情绪曲线。
- 结尾转化。

输出不应只是自然语言总结，而应是可迁移的结构卡或结构规则。

### 3.3 用户新内容输入

用户输入新创作目标。

输入内容包括：

- 主题或商品。
- 核心卖点。
- 可用素材。
- 目标人群。
- 平台场景。
- 期望时长。
- 语气和风格要求。

系统将这些信息整理成新内容画像，供结构迁移使用。

### 3.4 结构迁移与方案生成

系统把样例结构映射到新内容。

输出包括：

- 新脚本。
- 分镜方案。
- 字幕文案。
- 镜头建议。
- 封面标题。
- 样例结构与新方案结构的对应关系。

迁移结果必须能解释“样例中的结构如何迁移到了新内容”，避免退化成普通仿写。

### 3.5 结果展示与编辑优化

系统以可理解、可调整的方式展示结果。

需要支持：

- 样例结构和新方案结构对照。
- 时间线和分镜卡片展示。
- 多版本对比。
- 局部重写。
- 基于某个片段或产物继续生成分支。

每次重跑或返工都生成新版本，不静默覆盖旧结果。

## 4. 核心对象

第一版至少保留这些概念：

- `Workspace`：一次创作任务。
- `SampleVideo`：用户上传的样例视频。
- `MediaDerivative`：抽帧、封面、音频、标准化视频等派生产物。
- `VideoSegment`：带起止时间的视频片段。
- `StructureCard`：样例拆解出的结构卡。
- `ContentProfile`：用户新内容画像。
- `TransferPlan`：结构迁移方案。
- `GeneratedPlan`：新视频脚本、分镜、字幕、镜头建议。
- `Version`：重跑、返工、分支产生的新版本。
- `Trace`：运行链路和调试追踪。

核心产物必须能追踪来源、阶段、父级产物和版本关系。

## 5. UI 占位策略

第一版就保留未来入口，但可以显示空状态。

- 左侧“结构库”：后续支持多样例结构收藏。
- 左侧“版本历史”：第一版展示当前版本和重跑记录。
- 底部“字幕/语音轨”：第一版可为空，后续接 ASR/OCR。
- 底部“迁移映射轨”：第一版在生成方案后再展示。
- 右侧“AI 建议”：第一版展示处理状态，后续展示片段理解和迁移建议。
- 顶部“导出”：第一版置灰，后续接视频生产能力。

## 6. 第一版验收标准

- 上传视频后，工作台能显示处理状态、封面预览、基础信息和底部抽帧时间线。
- 处理完成后，每个抽帧都能对应时间戳，点击时间线帧能定位到预览区。
- 进入视频理解后，结构段能显示在时间线上，并能在右侧看到片段说明。
- 生成新方案后，能看到样例结构和新方案结构的对应关系。
- 重跑某个阶段时，旧结果仍可在版本历史中回看。
- 普通日志不展示完整素材内容，失败时能追踪到 run、stage、artifact。

## 7. 当前默认决策

- 第一版采用类剪映桌面工作台，不采用纯流程向导。
- UI 参考剪映/CapCut 的编辑器布局心智，但不复制品牌视觉。
- 当前重点是主链路和结构迁移表达，不做完整视频剪辑器。
- 第一版优先完成样例视频输入处理、基础预览、抽帧时间线和后续功能占位。
- 后续智能分析必须基于可追踪产物，不直接读取散乱临时文件。

## 8. 本阶段不做

- 不做完整视频剪辑器。
- 不做复杂多轨剪辑。
- 不做正式导出成片。
- 不做多样例结构组合。
- 不做效果评估闭环。
- 不把模型调用散落在 UI 或临时脚本里。
